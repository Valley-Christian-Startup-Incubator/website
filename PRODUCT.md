# Product

## Register

brand

## Users

Valley Christian School students in (or joining) the Startup Incubator — teen/young-adult builders, often first-time open-source contributors. They arrive to learn how to register a project into the incubator's GitHub organization so it can be maintained, deployed, and handed off to the next student. Secondary readers: mentors and faculty pointing students here.

## Product Purpose

This website is the registration portal for the Startup Incubator GitHub organization. SUI itself is a real accelerator that mentors students building impactful businesses; this site covers one specific job within it — walking a student through registering/handing off a project (a pull request to the `tap` registry repo). Success = a student understands what SUI is, then confidently opens their first registration PR.

## Brand Personality

Optimistic, energetic, encouraging. Voice is plainspoken and warm — a sharp older student showing you the ropes, not a corporate brochure. Three words: **bright, builder, generous**. It should feel like momentum and possibility, and make a first PR feel approachable rather than intimidating.

## Anti-references

- Cosmic/dark-gradient SaaS templates (the previous iteration — explicitly rejected).
- AI-slop gradients: `background-clip:text` gradient headings, multi-stop gradient buttons/logos, decorative glassmorphism.
- Generic dark "developer tool" aesthetic (navy + neon, terminal cosplay) — this is for students, not a devtools launch.
- Identical icon-card grids, tiny uppercase tracked eyebrows over every section, hero-metric templates.

## Design Principles

1. **Color is the voice.** Commit to flat, saturated brand color (saffron-led) — no gradients hedging the edges.
2. **A first PR should feel doable.** The registration walkthrough is the hero of the experience; clarity beats cleverness there.
3. **Honest scope.** Present SUI accurately as an accelerator; frame registration as the one thing this site does.
4. **Earned motion.** Motion fits what it reveals; never fade-on-scroll-everything by reflex.
5. **Made by students, for students.** Warmth and approachability over polish-for-polish's-sake.

## Accessibility & Inclusion

WCAG AA: body text ≥4.5:1, large text ≥3:1 — verified against the light/saffron surfaces. Full `prefers-reduced-motion` support (the Three.js field and all reveals degrade to static). Keyboard-navigable, visible focus rings, semantic landmarks.
